Neighborhood Preserving Embedding

The Neighborhood Preserving Embedding (NPE) algorithm can be considered as a linear approximation to the Locally Linear Embedding algorithm. Thus most of computation routines can be shared with LLE. NPE uses steps 1 and 2 of the Locally Linear Embedding and the main difference lies in the eigendecomposition based embedding.
According to the NPE algorithm embedding can be found from the solution of the following partial generalized eigenproblem: \[ R L R f = \lambda R R^T f \] where \( R \) is a matrix containing all feature vectors \( x_1 , \dots , x_N \) row-wise. The problem is solved for smallest eigenvalues \( \lambda_1, \dots, \lambda_t \) and its corresponding eigenvectors \( f_1, \dots, f_t \). Form embedding matrix such that \(i\)-th coordinate (\( i=1,\dots,N \)) of \(j\)-th eigenvector (\( j=1,\dots,t\) ) corresponds to \(j\)-th coordinate of projected \(i\)-th vector.

Tapkee implementation

To use the NPE algorithm put NEIGHBORHOOD_PRESERVING_EMBEDDING value as REDUCTION_METHOD to the parameter map. It is required to fill dimension of feature vectors to the parameter map as CURRENT_DIMENSION. You can also set number of neighbors \( k \) as NUMBER_OF_NEIGHBORS in the parameter map and the eigenproblem shift can be set using EIGENSHIFT key. To change the method of obtaining nearest neighbors use NEIGHBORS_METHOD (recommended value is COVER_TREE) key of the parameter map and to change the method of eigendecomposition use EIGEN_EMBEDDING_METHOD (recommended value is ARPACK).
